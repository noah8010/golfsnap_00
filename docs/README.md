## 문서 용도 상세

### 불변 문서 (수정 거의 없음)

#### `../CLAUDE.md`
- **AI 세션 시작 시 필수 읽기**
- 프로젝트 개요, 아키텍처, 절대 규칙
- 세션 시작/종료 체크리스트

#### `CODING_RULES.md`
- 파일 구조 및 네이밍 규칙
- 컴포넌트 작성 템플릿
- TypeScript, 스타일링, 상태 관리 규칙
- Git 커밋 규칙

#### `DESIGN_SYSTEM.md`
- 색상, 타이포그래피, 간격 시스템
- 컴포넌트 스타일 가이드
- 아이콘, 애니메이션 규칙
- 터치 인터랙션 가이드

### 가변 문서 (자주 업데이트)

#### `FEATURE_GUIDE.md`
- 전체 기능 목록 및 상세 설명
- 데이터 모델, 훅 사용법
- 기능 추가/수정 시 업데이트

#### `PROJECT_STATUS.md`
- 현재 완성도 및 진행 상황
- 매 세션 종료 시 업데이트

#### `TODO.md`
- 우선순위별 작업 목록
- 완료 항목 체크, 새 항목 추가
- 매 세션마다 업데이트

---

## 새 세션 시작 순서

```
1. ../CLAUDE.md 읽기 (필수)
2. PROJECT_STATUS.md 확인
3. TODO.md 확인
4. 필요 시 FEATURE_GUIDE.md 참조
```

---

## 문서 업데이트 가이드

### 언제 업데이트하나요?

| 상황 | 업데이트 문서 |
|------|-------------|
| 새 기능 추가 | `FEATURE_GUIDE.md`, `TODO.md` |
| 버그 수정 | `TODO.md` |
| 완성도 변경 | `PROJECT_STATUS.md` |
| 코드 규칙 추가 | `CODING_RULES.md` |
| 디자인 변경 | `DESIGN_SYSTEM.md` |

### 업데이트 체크리스트

```markdown
- [ ] 날짜 갱신 (최종 업데이트)
- [ ] 완성도 퍼센트 갱신
- [ ] 완료 항목 체크
- [ ] 새 TODO 항목 추가
```

## 문서 작성 규칙

1. **마크다운 형식** 사용
2. **한글** 우선 (기술 용어는 영어 허용)
3. **테이블** 적극 활용
4. **코드 블록** 사용하여 예시 제공
5. **최종 업데이트 날짜** 문서 상단에 표기
